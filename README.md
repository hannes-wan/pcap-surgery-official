## What problem does PCAP Surgery solve?

Wireshark is excellent for inspection, and editcap is useful for command-line trimming. But real support work often needs both: inspect the capture, remove sensitive packets, rewrite addresses, normalize timestamps, repair checksums, and export a clean PCAP without corrupting the evidence.

## The result

Use PCAP Surgery when you need a concrete result, not another generic tool window:

- Classic PCAP/CAP import and basic Ethernet PCAPNG enhanced packet support.
- Paged packet table with protocol, endpoint, port, text, and time filters.
- Drop/keep packets, subset export, timestamp scale/offset/normalize.
- Fixed-length byte edits, adjacent order edits, annotations, and rewrite rules.

## Who it is for

- Network engineers preparing PCAP evidence for vendors, tickets, and internal review.
- Security and support teams that must sanitize captures before sharing them outside the organization.
- Protocol developers building smaller regression fixtures from large real-world captures.

## Product capabilities

- Classic PCAP/CAP import and basic Ethernet PCAPNG enhanced packet support
- Paged packet table with protocol, endpoint, port, text, and time filters
- Drop/keep packets, subset export, timestamp scale/offset/normalize
- Fixed-length byte edits, adjacent order edits, annotations, and rewrite rules
- IPv4/TCP/UDP checksum repair for supported rewrites
- Export edited output as classic PCAP
